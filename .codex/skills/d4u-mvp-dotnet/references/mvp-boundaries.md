# D4U MVP Boundaries

Use this reference when deciding whether a requested change belongs in the first MVP.

## Include in MVP

- Email/password registration and login.
- Role-based users: STUDENT, SME, ADMIN.
- Student, SME, and Admin profiles.
- Student verification reviewed by Admin.
- Basic/Pro/Premium subscription plans for project limits and platform fee rate.
- Design categories.
- Projects: draft, open/private, publish, cancel, selected student.
- Project applications and offers.
- Escrow payment through sandbox/provider integration.
- Sketch and Final milestones.
- Submission files, watermark reference, review action.
- Revision request and invalid file report.
- Final approval, escrow release, disbursement, wallet credit.
- Payment methods and withdrawal requests.
- Basic dispute with evidence and admin decision stored on `disputes`.
- Rating after project completion.
- In-app notifications.
- Audit logs for important actions.

## Exclude From MVP

- Social login and SSO.
- Chat realtime.
- Portfolio builder and public profile gallery.
- Skill/software matching.
- AI recommendation.
- Automated subscription billing retry flows.
- Dispute appeal workflow.
- Separate dispute decision table.
- Reputation event ledger and warning workflow.
- Email/push notification delivery tracking.
- Advanced KYC.
- Contract/e-signature.
- Native mobile app.

## Rule of Thumb

If a feature is not needed to pass `MVP_D4U.md` section 11 checklist, keep it out unless the user explicitly changes MVP scope.
